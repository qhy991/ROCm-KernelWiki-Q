# [CK Tile] Prepare mixed batch-prefill FP8 KV contract

Open PR #3745 in [ROCm/composable_kernel](https://github.com/ROCm/composable_kernel/pull/3745).

**Author:** ysmkone  
**Updated:** 2026-06-11

## Description

This PR does not implement mixed-dtype CK Tile batch-prefill kernels. Its value for the wiki is the explicit contract boundary: BF16/FP16 Q plus FP8 K/V plus BF16/FP16 output is classified as an unsupported `mixed_q_fp8_kv` request rather than silently routing through a normal BF16/FP16 batch-prefill path.

The PR preserves the existing all-FP8 Q/K/V `fp8bf16` batch-prefill path and adds dispatcher/AOT validation tests around future explicit `q_data_type` and `kv_data_type` codegen configurations.
