# Support biased SwiGLU in MXFP4 MoE

Closed PR #3735 in [ROCm/composable_kernel](https://github.com/ROCm/composable_kernel/pull/3735).

**Author:** XiaobingSuper  
**Closed:** 2026-05-06

## Description

This PR adds bias-aware SwiGLU handling to MXFP4 MoE paths and fixes a split-K epilogue corner case. The proposed changes cover BNS and BPreShuffle device/gridwise paths, `swiglustep_and_mul` behavior for GPT-OSS-style clamp/alpha handling, and a correction so `k_batch == 1` stores instead of accumulating stale output.
