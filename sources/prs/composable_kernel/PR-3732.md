# [ck_tile/fmha] Fix sink un-mask under right-window and emit fp8bf16 batch_prefill sink kernels

Closed PR #3732 in [ROCm/composable_kernel](https://github.com/ROCm/composable_kernel/pull/3732). GitHub reports `merged_at: null`, so this page records it as closed/unmerged, not merged.

**Author:** jundaf2  
**Closed:** 2026-04-29

## Description

This PR proposed two CK Tile FMHA batch-prefill fixes for StreamLLM-style sink attention with sliding windows and fp8 K/V plus BF16 compute. The codegen change would emit `fp8bf16` batch-prefill kernels for `generic_with_sink` masking, so runtime requests using sink tokens would have a generated kernel variant instead of missing coverage.

The masking change targeted sink unmasking under a right-window constraint. In that case, the sink region and local window need to be combined carefully so sink tokens remain visible without incorrectly exposing tokens outside the right-window policy.
