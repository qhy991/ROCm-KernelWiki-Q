# [CK_TILE] async trload for fmha 192/128 in mi355

Closed PR #3729 in [ROCm/composable_kernel](https://github.com/ROCm/composable_kernel/pull/3729).

**Author:** gino-lu  
**Closed:** 2026-05-27

## Why It Matters

The PR is a compact signal that high-performance FMHA on CDNA4 depends on more than the math tile. Attention head dimensions such as 192 and 128 stress LDS/register staging, and async transpose-load support changes how Q/K/V tiles are fed into the MFMA loop.

Even though the PR closed, the changed files point at the relevant design boundary: code generation selects FMHA variants, while architecture helpers expose whether an async transpose-load path is legal.
