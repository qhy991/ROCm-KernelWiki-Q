# [CK_TILE] MX GEMM, non-preshuffled and RCR layout

Closed PR #3709 in [ROCm/composable_kernel](https://github.com/ROCm/composable_kernel/pull/3709).

**Author:** samremes  
**Closed:** 2026-02-17

## Why It Matters

This PR is useful evidence for CDNA4 low-precision GEMM design even though it closed without merge. It shows the implementation surface needed for MX GEMM variants: example programs, scale-pointer abstractions, GEMM-MX kernel plumbing, and an async AG/BG/CR pipeline.

The key wiki lesson is that block-scaled GEMM is not just a dtype switch. Layout choices such as preshuffled versus non-preshuffled B and RCR ordering change where scale metadata is read, how operands are staged, and how much register/LDS pressure the pipeline carries.
