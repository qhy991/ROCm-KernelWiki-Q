# [CK_TILE] fix(fmha): clamp paged KV lookups in batch prefill

Closed PR #3733 in [ROCm/composable_kernel](https://github.com/ROCm/composable_kernel/pull/3733).

**Author:** zhenhantech  
**Closed:** 2026-05-27

## Description

FMHA batch prefill can prefetch V pages beyond the valid KV sequence length. For SGLang-style page tables, `kv_indptr` defines the valid logical page-table range, so a V prefetch can reach a one-past-end `page_idx` lookup.

The proposed fix exposes the mask's KV sequence length through `GetXTotal()` and passes that limit into `load_physical_pages()`, clamping each page-table lookup token to the last valid KV token before reading `page_idx`.
