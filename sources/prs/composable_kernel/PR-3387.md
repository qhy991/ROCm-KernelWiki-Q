# [CK_TILE] Minor splitk bugfix for gemms and conv

Merged PR #3387 in [ROCm/composable_kernel](https://github.com/ROCm/composable_kernel/pull/3387).

**Author:** jakpiase
**Merged:** 2025-12-23

## Description

> Currently our gemm and some conv kernels that internally run gemms are assuming that every threadblock except the last one will do the same amount of work, and splitk is internally implemented by shifting pointers. It works almost always, but there is an edge case, if `splitk < grid` then some threadgroups shouldn't do any work, i.e. should have `numloops = 0`.

See the PR for full details including code changes and review discussion.
