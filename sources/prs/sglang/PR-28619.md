# Fix Qwen MoE precision issue with PP and all-reduce fusion

Open PR #28619 in [sgl-project/sglang](https://github.com/sgl-project/sglang/pull/28619). It is open and unmerged as of capture time.

**Author:** 1e4ves  
**Opened:** 2026-06-18  
**Status verified:** open on 2026-06-19

## What Changes

The PR fixes a Qwen MoE precision issue when pipeline parallelism, tensor parallelism, and FlashInfer all-reduce fusion are used together:

- SGLang can defer a post-expert all-reduce and fuse it into the next layer's attention-prepare path.
- The deferred reduction is tracked by `_sglang_needs_allreduce_fusion`.
- At a pipeline-parallel boundary, hidden states are wrapped into proxy tensors and the Python-side marker is not preserved.
- The PR flushes the pending MoE or expert-parallel all-reduce before returning the PP proxy tensor.

## Kernel Relevance

This is a collective scheduling fix, not a new MoE kernel. It is useful evidence for a serving-kernel wiki because it documents a correctness contract for fused collectives: if a reduction is deferred into a later kernel/phase, the framework must preserve or flush that obligation across graph, stage, and process boundaries.

Do not describe this as improving MoE compute performance; its claim is precision correctness under PP+TP all-reduce fusion.
