# [minimax-m3] Split 4/4: model + VL + glue + function-call + fp8 quant + generic infra

**Status verified:** open on 2026-06-19

## What Changes

This is the final integration PR in a four-part MiniMax-M3 split and depends on #28712, #28713, and #28714. It carries the broad model/runtime changes:

- MiniMax-M3 and MiniMax-M3-VL model definitions, configs, multimodal processing, function-call parsing, and scheduler/server glue.
- Attention registry and `minimax_sparse_backend` integration.
- FP8 quantization updates in `fp8.py`, `fp8_kernel.py`, `fp8_utils.py`, and `mxfp8_block_convert.py`.
- Generic MoE infrastructure changes in `ep_moe`, `topk`, `token_dispatcher`, `layernorm`, and `moe_fused_gate`.
- JIT per-token-group quantization and post-reorder DeepGEMM-related hooks.

## Kernel Relevance

Use this PR as the high-level MiniMax-M3 integration source, not as a single kernel patch. It is useful because it ties earlier sparse-attention and norm/MoE kernel foundations into the model path and makes some generic quant/MoE infrastructure changes unconditionally visible.

The PR itself warns that several changes affect generic default paths rather than being fully MiniMax-guarded. Wiki references should therefore distinguish model glue from actual kernel implementations in the earlier split PRs.
