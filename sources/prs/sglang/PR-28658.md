# [AMD] Fuse shared-expert sigmoid + bf16->fp32 cast into the MoE append kernel

Open PR #28658 in [sgl-project/sglang](https://github.com/sgl-project/sglang/pull/28658). It remains open as of this capture.

**Author:** yichiche  
**Opened:** 2026-06-18  
**Status verified:** open on 2026-06-19

## What Changes

The PR targets the AITER shared-expert-fusion path for Qwen2-MoE-style serving. Before the change, preparing shared-expert routing launched three small kernels per MoE layer per decode step:

- Sigmoid on shared-expert logits.
- BF16-to-FP32 cast for shared weights.
- Append shared experts with weights.

The proposed fused append kernel performs the sigmoid and cast internally, reducing the sequence from three launches to one for the compatible path.

## Kernel Relevance

This is a small but important serving-kernel pattern: elementwise glue around MoE can dominate decode latency when tensors are small and launch-bound. Fusing the glue into the append kernel:

- Avoids writing/reading intermediate shared weights.
- Removes two kernel launches per MoE layer in decode.
- Keeps fusion close to the consumer kernel, where routing layout is already known.
