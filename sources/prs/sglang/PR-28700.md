# [AMD][Perf] Fuse QK RMSNorm + 3D mRoPE + KV-cache store into single aiter op

Open PR #28700 in [sgl-project/sglang](https://github.com/sgl-project/sglang/pull/28700). It is an open optimization proposal for SGLang, not a released default.

**Author:** yichiche  
**Opened:** 2026-06-19  
**Status verified:** open on 2026-06-19

## What Changes

The PR targets Qwen3.5 full-attention decode on ROCm/AITER. The existing `forward_prepare` path launches separate kernels for:

- QK RMSNorm and gate split.
- 3D mRoPE rotary embedding.
- Paged KV-cache store.

The proposed path calls AITER `fused_qk_norm_mrope_3d_cache_pts_quant_shuffle`, collapsing those steps into one HIP kernel when the model and tensor-parallel configuration match the Qwen3.5-397B-A17B-MXFP4 serving path.

## Kernel Relevance

This is a strong serving-kernel fusion example because it removes launch overhead and intermediate memory traffic around attention setup rather than the attention matmul itself. The PR's measured motivation is decode-step latency: three micro-kernels per full-attention layer are individually small but repeated every token.

For wiki purposes, treat this as a fused prepare/op-store contract:

- RMSNorm output cannot be considered independently from RoPE layout.
- RoPE output is written directly into the KV-cache layout expected by paged attention.
- Dispatch guards are part of correctness because the fused AITER op encodes shape, quantization, and cache assumptions.

## Wiki Use

- `kernel-rmsnorm-rocm`: norm + RoPE + KV-cache fusion evidence.
- `kernel-paged-prefill-attention-rocm`: KV-cache write contract adjacent to paged attention.
