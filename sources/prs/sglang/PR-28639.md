# [feat] add ag_gemm and moe_rs overlap kernels for dsv4 prefill

Open PR #28639 in [sgl-project/sglang](https://github.com/sgl-project/sglang/pull/28639). It is open and unmerged as of capture time.

**Author:** Zqy11  
**Opened:** 2026-06-18  
**Status verified:** open on 2026-06-19

## What Changes

This PR adds two opt-in communication/computation overlap kernels for DeepSeek V4 prefill context parallelism:

- Fused AllGather + FP8 quantization + shared-expert GEMM, implemented with `torch.symmetric_memory` and Triton.
- Fused MoE topk-reduce + shared expert addition + ReduceScatter.
- A new environment gate `SGLANG_OPT_USE_TORCH_SYMM_MEM_FUSED_KERNEL` controls the path.
- `TorchSymmMemCommunicator` gains lazy context helpers and CP-mode activation.
- DeepSeek V2/V4 and MoE runner paths integrate the fused operations while skipping standalone CP AllGather when active.

## Kernel Relevance

This is not a ROCm PR; it targets the DeepSeek V4 prefill path and symmetric-memory communication overlap. It is still useful for kernel-wiki comparison because it shows a modern serving-kernel pattern: move collectives and quantization into the same scheduling envelope as GEMM/reduction, so Copy Engine transfers and SM work overlap instead of serializing around MoE prefill.

For ROCm notes, treat it as a design analogue rather than ROCm evidence unless a HIP/ROCm implementation is later added.
