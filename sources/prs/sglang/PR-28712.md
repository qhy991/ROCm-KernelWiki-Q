# [minimax-m3] Split 1/4: sparse attention ops + JIT kernels + config foundation

Open PR #28712 in [sgl-project/sglang](https://github.com/sgl-project/sglang/pull/28712). It is an additive foundation PR and is not merged as of capture time.

**Author:** JustinTong0323  
**Opened:** 2026-06-19  
**Status verified:** open on 2026-06-19

## What Changes

This is part 1 of a split MiniMax-M3 integration. The PR is intentionally additive and includes kernel infrastructure rather than wiring the full model forward path:

- `minimax_sparse_ops` decode, prefill, and naive sparse-attention implementations with tests.
- JIT MiniMax/M3 kernels for QK-norm/RoPE, RMSNorm, SwiGLU, per-token quantization, store-KV-index, and decode top-k.
- A generic `moe_topk_sigmoid` JIT kernel.
- ROCm gfx95 MXFP8 MoE configuration and quantization helpers.
- Benchmarks and registered tests for sparse attention and MiniMax kernels.

## Kernel Relevance

This PR is high value because it shows the kernel surface area required before a new sparse MoE serving model can be enabled:

- Attention needs both decode and prefill sparse kernels, plus page/table helpers.
- Normalization/RoPE/store helpers are separate JIT kernels unless later fused.
- MoE requires both routing/top-k kernels and backend-specific MXFP8 GEMM configs.
- Keeping the PR dead-on-main reduces integration risk while still landing reusable kernels.

## Wiki Use
