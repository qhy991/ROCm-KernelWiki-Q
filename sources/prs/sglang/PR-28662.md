# [Fix] compressed-tensors block FP8: requantize weight scales to UE8M0 for DeepGEMM on Blackwell

Open PR #28662 in [sgl-project/sglang](https://github.com/sgl-project/sglang/pull/28662). It is open and unmerged as of capture time.

**Author:** Jiminator  
**Opened:** 2026-06-18  
**Status verified:** open on 2026-06-19

## What Changes

The PR fixes a compressed-tensors block-FP8 path for Blackwell DeepGEMM:

- `CompressedTensorsW8A8Fp8` with `BLOCK` strategy previously kept raw float32 block weight scales.
- DeepGEMM on Blackwell quantizes activations to UE8M0 scales and expects weight scales to be UE8M0-packed as well.
- The PR mirrors the existing `Fp8LinearMethod` behavior by requantizing weight scales in-place when the active runner is `deepgemm_w8a8_block_fp8_linear_with_fallback`.
- A manual Blackwell regression test is added and skips unless SM100 plus DeepGEMM are available.

## Kernel Relevance

This is a scale-layout compatibility fix, not a new GEMM kernel. It is important because block-FP8 kernels often fail silently or catastrophically when activation scale format and weight scale format differ. The PR records the backend contract: for this DeepGEMM path, both activation and weight scale metadata must be UE8M0-compatible.

For ROCm use, cite it only as a quantized-GEMM scale-contract analogue unless a ROCm backend adopts the same UE8M0 packing convention.
