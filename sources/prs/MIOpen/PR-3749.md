## Description

> This PR is an attempt to bring order and put things in their proper places. The cause of the problem is the `Composable Kernel` library - more precisely, a modern version of it which is located in a separate repository, as well as two earlier versions deeply embedded within the `MIOpen`. The earliest one is `Static CK` or `static_ck`, the later one is `Legacy CK` or `legacy_ck`, and the modern version is `CK`. Originally no one called them that - they were all simply known as `Composable Kernel`

See the PR for full details including code changes and review discussion.
