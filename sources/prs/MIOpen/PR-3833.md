# Fix NCHW precision issues for fused CBA/CA solvers

Merged PR #3833 in [ROCm/MIOpen](https://github.com/ROCm/MIOpen/pull/3833).

**Author:** SamuelReeder
**Merged:** 2025-07-07

## Description

> This pull request achieves support of NCHW & NCDHW layout for fused CBA/CA solvers. The prior issue was that the workspace was not entirely implemented, and there was a `nullptr` in place of it. When a convolution with supposedly NCHW tensors was performed, the convolution was actually being performed on NHWC tensors, because the tensors were never transformed. Therefore, the computation lacked precision as it was using misarranged data.

See the PR for full details including code changes and review discussion.
