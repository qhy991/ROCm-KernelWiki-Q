# Enable less restrictive layout selection

Merged PR #3864 in [ROCm/MIOpen](https://github.com/ROCm/MIOpen/pull/3864).

**Author:** BrianHarrisonAMD
**Merged:** 2025-07-09

## Description

> The previous less restrictive changes were causing some cases where C == 1, or H & W == 1 to have a changed layout selection choice. This layout choice could impact solution selection in several ways, leading to new defects, or performance issues being found. This change attempts to restore the previous layout choices, and extends to allow additional cases where less restrictive checks are still valid if no strict layout is applicable to all tensors.

See the PR for full details including code changes and review discussion.
