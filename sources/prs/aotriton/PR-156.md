# Fix Windows build for gfx942 affine kernels: use wide string for pstring_view

Merged PR #156 in [ROCm/aotriton](https://github.com/ROCm/aotriton/pull/156).

**Author:** jammm
**Merged:** 2026-03-03

## Description

> On Windows, pstring_view is std::wstring_view (since std::filesystem::path::value_type is wchar_t). The validator.cc template emitted narrow R-string literals for package_path, causing 'no viable overloaded =' errors when building for gfx942. The autotune_table_entry.cc template already had the correct #if defined(_WIN32) / LR-string guard; this applies the same fix to validator.cc.

See the PR for full details including code changes and review discussion.
