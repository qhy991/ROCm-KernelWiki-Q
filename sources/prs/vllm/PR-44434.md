# enable shared expert fusion for Qwen3.5

## Summary
Enables Fused Shared Expert (FSE) support for Qwen3.5 models on the ROCm AITER backend, fixing a bug where checkpoint weight names failed to map to the fused expert slot.

## Fix Details
In Qwen3.5, the MoE block overrides weight loading logic to handle both fused `gate_up_proj` and separate projection formats. When FSE is enabled via `VLLM_ROCM_USE_AITER_FUSION_SHARED_EXPERTS=1`, the shared expert must be treated internally as a fused expert mapped to slot `E+1`. The PR correctly remaps `shared_expert.*` weights to `experts.{num_experts}.*` and resets `is_fused_expert` so the shared expert enters the appropriate loading path instead of failing silently.

## Impact
On a Qwen3.5-397B-A17B model on MI355X hardware:
- Maintains identical GSM8K 5-shot accuracy.
- Improves TP4 (Tensor Parallel 4) serving throughput by **+25.9%** (from 497 tok/s to 626 tok/s at concurrency 8).
