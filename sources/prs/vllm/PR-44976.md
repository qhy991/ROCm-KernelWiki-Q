# [ROCm][Kernel][AITER] BlockScale FP8 SplitK zero-init fusion

Open PR #44976 in [vllm-project/vllm](https://github.com/vllm-project/vllm/pull/44976).

**Author:** samremes  
**Updated:** 2026-06-18

## Description

ROCm AITER block-scale FP8 GEMM needs a zeroed output buffer before Split-K accumulation. This PR removes a standalone `Y.zero_()` fill kernel by folding zero-init into the upstream FP8 quantization or RMSNorm producer, then rewrites the GEMM to a Split-K variant with `y_is_zeroed=True`.
