# [Bugfix][ROCm] Fix OOB query read in paged_attention_rocm for head_size < 128

Open PR #40745 in [vllm-project/vllm](https://github.com/vllm-project/vllm/pull/40745).

**Author:** Concurrensee  
**Updated:** 2026-05-15

## Description

The ROCm paged-attention kernels in `csrc/rocm/attention.cu` assumed a 128-wide head and mapped 64 wavefront lanes to 16 query chunks. For `head_size=64`, only 8 chunks exist, so the upper half of the wavefront can read up to 64 bytes past the query allocation.

## Changed Files

- `csrc/rocm/attention.cu`

## Wiki Use

- `kernel-paged-prefill-attention-rocm`: head-size guards and wavefront lane mapping.
- `hw-wavefront`: 64-lane assumptions can become correctness hazards when porting kernels.
