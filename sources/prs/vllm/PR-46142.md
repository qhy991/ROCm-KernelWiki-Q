# Improve w_mxfp4_a_fp8 emulation support on MI300

## Summary
The Open Compute Project Microscaling Formats (OCP MX) provide extreme quantization (e.g., MXFP4). On hardware like MI250/MI300/MI325, this must be accurately emulated if hardware support is incomplete. This PR fixes the MoE emulation code path for `W-MXFP4-A-FP8` (Weight MXFP4, Activation FP8).

## Fix Details
- **Wrongful Dispatch Fixed**: The `UNFUSED_TRITON` backend was silently skipping input quantization for MXFP4 MoE matrices. This PR forces the correct emulation code path so that activations are quantized exactly as the models require.
- **Double Call Removed**: Removed a redundant `moe_kernel_quantize_input` call for the `a13` intermediate state in `fused_moe/experts/ocp_mx_emulation_moe.py` to prevent double quantization drift.
