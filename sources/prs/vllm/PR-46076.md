# [Attention][DSA] support dcp for FLASHINFER_MLA_SPARSE

## What Changes

This PR extends the sparse MLA attention path so `FLASHINFER_MLA_SPARSE` can run with decode-context parallelism (DCP):

- Adds DCP localization tests for the attention indexer.
- Updates `mla_attention.py` and the `flashinfer_mla_sparse.py` backend to pass localized metadata correctly.
- Extends the MLA indexer implementation for DCP-aware sparse attention routing.
- Documents the backend capability in `docs/design/attention_backends.md`.

The PR description validates GLM-5.2-FP8 serving with `--tensor-parallel-size 8` and `--decode-context-parallel-size 4`, reporting a larger GPU KV cache token capacity than TP-only serving.

## Kernel Relevance

This is framework/backend integration around sparse MLA attention rather than a new low-level kernel. It matters for ROCm attention notes because DCP changes which KV/page metadata each rank owns. A sparse/paged attention backend must localize indices before invoking the kernel, otherwise the compute kernel can receive globally meaningful but rank-local invalid offsets.
