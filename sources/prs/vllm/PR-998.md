# Pad unquantized weight stride off the gfx11x 4096 B cliff

## Summary
On RDNA3 (gfx11x/gfx1151), unquantized GEMMs where the weight row stride is a multiple of 4096 bytes hit severe L2 cache aliasing/collisions, plummeting effective memory bandwidth.

## Solution
By padding the weight row stride by exactly one cache line (+128 Bytes), the collision is completely avoided. The `ldb` (leading dimension) is padded once at load time in `UnquantizedLinearMethod.process_weights_after_loading`. This results in zero runtime arithmetic overhead but yields a massive ~13.6% TTFT improvement for OpenVLA-7B.
