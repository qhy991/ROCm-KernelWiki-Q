# Pad hybrid MoE's GEMMs weight row stride off the gfx11x cache cliff

## Summary
The hybrid W4A16 MoE expert weights are stored as skinny `[E, N, K//8]` int32. At specific shapes (like Qwen3-A3B), the per-row byte stride lands on exactly 1024 B, and the per-expert stride on 1 MB. This causes severe RDNA3 L2/MALL cache aliasing.

## Solution
Analogous to the unquantized dense linear layer optimization, the MoE buffer is padded by +32 int32 (+128 Bytes). The triton prefill kernel and the HIP decode `wvSplitK` kernel automatically recalculate the correct padded stride, recovering up to 13ms of prefill latency.
