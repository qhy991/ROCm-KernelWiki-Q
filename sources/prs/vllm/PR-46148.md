# [ROCm][CI] Only require q_scale==1.0 for fp8 query in RocmAttention

Open PR #46148 in [vllm-project/vllm](https://github.com/vllm-project/vllm/pull/46148). It is open and unmerged as of capture time.

**Author:** stefankoncarevic  
**Opened:** 2026-06-19  
**Status verified:** open on 2026-06-19

## What Changes

The PR changes the ROCm attention guard in `RocmAttentionImpl.forward()`:

- The old guard asserted `layer._q_scale_float == 1.0` whenever the KV cache dtype was quantized.
- The PR scopes that requirement to the case where the query tensor is actually FP8.
- The motivating ROCm MHA path, `chunked_prefill_paged_decode`, runs with a full-precision query and FP8 KV cache; in that case `q_scale` is not consumed by the kernel.

## Kernel Relevance

This is an attention compatibility fix rather than a new kernel. It is still valuable for the ROCm kernel wiki because it defines a scale contract for FP8-KV attention:

- FP8 K/V cache does not imply FP8 query.
- Query dequant scale should be validated only when the query operand is quantized.
- Over-broad framework assertions can prevent valid BF16-query/FP8-KV attention from reaching the ROCm backend.

The PR specifically mentions startup failures for FP8-KV checkpoints with non-1.0 `q_scale`, including `nm-testing/Llama-3.2-1B-Instruct-FP8-KV`.
