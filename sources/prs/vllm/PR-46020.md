# [Attention Backend] add HPC-Ops Attention backend

Open PR #46020 in [vllm-project/vllm](https://github.com/vllm-project/vllm/pull/46020). It is open and unmerged as of capture time.

**Author:** thisjiang  
**Opened:** 2026-06-18  
**Status verified:** open on 2026-06-19

## What Changes

This PR adds an optional attention backend backed by Tencent's HPC-Ops library:

- Registers `hpc_attn.py` as a new vLLM v1 attention backend.
- Adds `vllm/model_executor/layers/hpc/` with module wrappers and RoPE/norm support.
- Wires model-loader metadata and Hunyuan V3 model code to support the backend path.
- Adds design documentation and test coverage that skips when HPC-Ops is not installed.

## Kernel Relevance

This PR is relevant as a backend-integration source, not as evidence of a ROCm-native kernel unless the external HPC-Ops implementation is separately inspected. It shows how vLLM admits an external attention operator library: configuration registration, model-level routing, helper modules for fused prepare work, and backend tests.

Because the PR is not ROCm-specific in its title or files, use it carefully in the ROCm wiki: cite it for attention backend extensibility and fused RoPE/norm-adjacent contracts, not as proof of a ROCm kernel optimization.
