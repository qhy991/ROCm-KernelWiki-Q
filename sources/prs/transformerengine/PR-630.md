# gfx1250 mxfp8 gemm: add NN/NT transpose workaround

Open PR #630 in [ROCm/TransformerEngine](https://github.com/ROCm/TransformerEngine/pull/630).

**Author:** matthiasdiener  
**Opened:** 2026-06-16  
**Last updated:** 2026-06-16  
**Base branch:** `mdiener/loosen-gfx1250-mxfp8-k-restriction`

## Status

GitHub reports this PR as open and not merged as of capture on 2026-06-19. It changes 3 files with about 253 additions and 13 deletions.

## What Changed

- Adds small HIP transpose kernels for byte-scale buffers and typed output buffers in `transformer_engine/common/gemm/rocm_gemm.cu`.
- Adds MXFP8 scale conversion between colwise and rowwise presentations, including handling for swizzled input/output scale layouts.
- Introduces temporary-buffer handling so unsupported NN/NT presentations can be converted into a supported GEMM route and then converted back if needed.
- Extends `test_cublaslt_gemm.cu` to cover the workaround path.

## Kernel Relevance

This is evidence of a layout gap in the gfx1250 MXFP8 GEMM path rather than a final architecture capability claim. It shows that NN/NT support may require explicit transpose and scale-layout conversion around the GEMM call.
