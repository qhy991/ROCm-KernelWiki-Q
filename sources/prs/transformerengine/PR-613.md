# CK MXFP8 Group Gemm gfx1250 Enablement

Merged PR #613 in [ROCm/TransformerEngine](https://github.com/ROCm/TransformerEngine/pull/613).

**Author:** aris134  
**Merged:** 2026-06-15  
**Base branch:** `dev`

## Status

GitHub reports this PR as closed and merged on 2026-06-15. It changed 12 files with about 1200 additions and 43 deletions.

## What Changed

- Adds a CK MXFP8 grouped GEMM integration path in `transformer_engine/common/gemm/ck_grouped_gemm/ck_mx_grouped_gemm.cpp`.
- Adds runtime architecture detection for gfx1250 support and dispatches through the ROCm grouped GEMM path rather than treating it as a CUDA-only grouped GEMM case.
- Adds scale preshuffle helpers for gfx1250 MXFP8 scale buffers, including a guard that ties `KScale` to 4-scale groups, equivalent to original K being a multiple of 128 when the scale block is 32.
- Adds `tests/cpp/operator/test_ck_grouped_mxfp8.cu` and extends PyTorch grouped-linear test coverage notes for MXFP8 with `NVTE_ROCM_ENABLE_MXFP8=1`.
- Adds a `rocm_libraries` submodule path for the Composable Kernel definitions needed by this TE integration.

## Kernel Relevance

This is high-value evidence for ROCm TransformerEngine adopting CK MXFP8 grouped GEMM as an integration surface. The PR is explicitly gfx1250-oriented, so it should be cited as RDNA4/gfx1250 integration evidence and not as a CDNA4 scaled-MFMA performance result.
