# add MXFP8 pre-swizzling for gfx1250 GEMM

**Merged:** 2026-05-21  
**Base branch:** `gfx1250`

## Status

GitHub reports this PR as closed and merged on 2026-05-21. It changed 10 files with about 524 additions and 36 deletions.

## What Changed

- Introduces the initial gfx1250 MXFP8 GEMM pre-swizzling path on the `gfx1250` branch.
- Adds `test_swizzle.cu` coverage and extends `test_cublaslt_gemm.cu` to exercise scale-layout behavior.
- Wires PyTorch and JAX GEMM extensions so frontends can pass the required scale-swizzle metadata to the backend.
- Updates `MXFP8Tensor` and quantizer plumbing so GEMM-swizzled scale buffers are represented in TransformerEngine's tensor abstractions.

## Kernel Relevance

This is the original PR behind the later dev-branch PR #605. It is useful for tracing the feature lineage and for understanding why scale metadata layout became part of the gfx1250 MXFP8 GEMM contract.
