# HipKittens MXFP8 GEMM Support

Open PR #566 in [ROCm/TransformerEngine](https://github.com/ROCm/TransformerEngine/pull/566).

**Author:** alextmagro  
**Opened:** 2026-04-28  
**Base branch:** `dev`

## Status

GitHub reports this PR as open as of 2026-06-19. It changes 15 files with about 1,189 additions and 62 deletions.

## What Changed

- Adds `3rdparty/hipkittens` as a submodule and wires it into the ROCm GEMM build.
- Adds `transformer_engine/common/gemm/kittens/mxfp8_gemm.cpp` and `mxfp8_gemm.h` for a HipKittens MXFP8 GEMM backend.
- Extends PyTorch and JAX GEMM extension plumbing so frontends can dispatch to the new backend.
- Adds C++ GEMM tests for the new path.
- PR description says the path only builds for `gfx950` and requires `M / 256` and `N / 256`; treat those as source-reported constraints while the PR remains open.

## Kernel Relevance

This is a direct MXFP8 GEMM implementation proposal for CDNA4/gfx950. It is especially relevant for backend-comparison notes because the PR claims performance advantages over hipBLASLt plus extra epilogues, but the page should describe those as author-reported until merged and independently benchmarked.
