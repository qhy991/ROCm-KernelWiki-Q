# Full MXFP4 Training Recipe

Merged PR #537 in [ROCm/TransformerEngine](https://github.com/ROCm/TransformerEngine/pull/537).

**Merged:** 2026-04-28  
**Base branch:** `dev`

## Status

GitHub reports this PR as closed and merged on 2026-04-28. It changed 32 files with about 4087 additions and 66 deletions.

## What Changed

- Introduces `nvte_cast_transpose_mxfp4_fused_shuffle`, a fused HIP path for MXFP4 cast+transpose with optional Hadamard transform and GEMM-oriented memory-layout shuffling.
- Adds an `fp4_gemm_handler` that dispatches A4W4 GEMM calls to AITER with expected layouts.
- Adds MXFP4 weight caching in `Linear.forward()` and `LayerNormLinear.forward()` using `MXFP4TensorStorage` across forward passes.
- Adds the `quantization_mxfp4.py` custom recipe and frontend wiring for PyTorch modules and quantization APIs.
- Adds C++ and Python tests for cast, transpose, GEMM exactness, module exactness, and quantization exactness.

## Kernel Relevance

This is the broadest TransformerEngine source for MXFP4 training on ROCm. It combines data preparation kernels, quantization metadata, AITER GEMM dispatch, and framework-level weight caching. The PR test plan mentions gfx950 and gfx942 logs plus end-to-end MXFP4 training, but the kernel claims should stay scoped to source-reported ROCm TransformerEngine behavior.
