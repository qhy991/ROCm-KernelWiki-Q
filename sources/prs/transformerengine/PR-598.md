# Mxfp8 grouped and multi quantize

Open PR #598 in [ROCm/TransformerEngine](https://github.com/ROCm/TransformerEngine/pull/598).

**Author:** alextmagro  
**Opened:** 2026-05-27  
**Base branch:** `dev`

## Status

GitHub reports this PR as open as of 2026-06-19. It changes 15 files with about 1,555 additions and 528 deletions.

## What Changed

- Introduces grouped and multi-quantize kernels for MXFP8.
- Adds `bench_group_quantize_mxfp8.cpp` and C++ tests for grouped and multi quantization.
- Refactors the ROCm MXFP8 quantize kernel body into `rocm_quantize_mxfp8_body.inc` to reduce repeated code.
- Updates dispatch and PyTorch extension plumbing in `quantize.cuh`, `cast.cpp`, and `mxfp8_tensor.py`.
- PR description notes nontemporal-store improvements and also says the grouped kernel still needs later optimization.

## Kernel Relevance

Although this is not a GEMM kernel itself, MXFP8 quantization layout and throughput are part of the end-to-end GEMM/MoE performance envelope. Treat this page as quantize-kernel support evidence, not as a direct GEMM implementation or final optimized grouped-quantize result while the PR is open.
