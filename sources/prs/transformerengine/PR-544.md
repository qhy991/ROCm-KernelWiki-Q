# [TE] Improve backward performance for CK Tile FP8 Group GEMM

**Base branch:** `dev`

## Status

GitHub reports this PR as closed and merged on 2026-04-26. It changed 3 files with about 79 additions and 24 deletions.

## What Changed

- Re-expresses WGRAD grouped GEMM in alternate FP8 storage/layout encoding through `columnwise_data`.
- Extends a layout rewrite already used for DGRAD to WGRAD.
- Steers CK away from less favorable transposed/shuffle handling in the FP8 pipeline while preserving the math.
- PR description reports a 3.15x median performance improvement for the affected backward path; treat this as source-reported benchmark evidence.

## Kernel Relevance

This is a focused CK Tile FP8 grouped-GEMM optimization for TransformerEngine backward training. It is useful evidence that operand presentation and layout canonicalization can be as important as the underlying GEMM kernel when trying to keep WGRAD on the fast CK path.
