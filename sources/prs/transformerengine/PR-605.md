# add MXFP8 pre-swizzling for gfx1250 GEMM (#568)

Merged PR #605 in [ROCm/TransformerEngine](https://github.com/ROCm/TransformerEngine/pull/605).

**Author:** matthiasdiener  
**Merged:** 2026-06-15  
**Base branch:** `dev`

## Status

GitHub reports this PR as closed and merged on 2026-06-15. It changed 12 files with about 694 additions and 39 deletions.

## What Changed

- Cherry-picks the gfx1250 MXFP8 pre-swizzling work from PR #568 into `dev`.
- Extends the ROCm MXFP8 GEMM path so scale tensors can be marked with GEMM-swizzled scale layout before dispatch.
- Updates PyTorch and JAX GEMM extension glue so MXFP8 scale-layout metadata reaches the common backend.
- Adds or updates C++ tests around `test_cublaslt_gemm.cu`, `test_swizzle.cu`, and production-like MXFP8 GEMM shapes.
- Adds a ROCm dequantization header change in `rocm_dequantize_mxfp8.cuh`, tying the feature to the MXFP8 data path rather than only a test-only swizzle helper.

## Kernel Relevance

The core kernel pattern is scale-layout preparation before GEMM. It documents that gfx1250 MXFP8 GEMM requires explicit scale pre-swizzling for the backend layout. Do not use this PR as evidence for CDNA4/gfx950 behavior; the PR title, tests, and implementation target gfx1250.
