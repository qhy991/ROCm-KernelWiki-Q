## What Changed

- Adds a workaround in `transformer_engine/pytorch/cpp_extensions/gemm.py` for intermittent incorrect results in NVFP4 backward GEMMs.
- Updates distributed numerics coverage in `tests/pytorch/distributed/run_numerics.py`.
- PR description reports the issue on `gfx942` and `gfx950`, so this page maps to CDNA3 and CDNA4.
- The motivating reproducer uses TransformerEngine distributed numerics with `--quantization nvfp4`.

## Kernel Relevance

The important kernel lesson is correctness gating: block-scaled FP4 GEMM paths may need backend or dispatch workarounds for backward training GEMMs on specific ROCm targets. This is not a speedup PR; it records a production correctness constraint.
