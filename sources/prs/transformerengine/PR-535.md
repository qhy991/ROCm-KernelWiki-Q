# MXFP4: Add GEMM kernel tuning and MXFP4Quantizer.copy()

Merged PR #535 in [ROCm/TransformerEngine](https://github.com/ROCm/TransformerEngine/pull/535).

**Author:** sarthak-amd  
**Merged:** 2026-04-09  
**Base branch:** `feature/mxfp4-recipe-210`

## Status

GitHub reports this PR as closed and merged on 2026-04-09. It changed 2 files with about 53 additions and 8 deletions.

## What Changed

- Adds CSV-based MXFP4 GEMM kernel selection through AITER `get_GEMM_config(M, N, K)`.
- Gates tuning with `NVTE_FP4_GEMM_TUNING`, enabled by default, and adds `NVTE_FP4_LOG_GEMM_SHAPES` for shape logging.
- Falls back to layout-based defaults when tuning is disabled or no CSV entry matches the GEMM shape.
- Adds `MXFP4Quantizer.copy()` so code paths that clone quantizers do not fail with `AttributeError`.
- PR test notes report Llama2 SFT MXFP4 convergence on 1x8 MI355X; treat that as source-reported recipe validation, not independent benchmark reproduction.

## Kernel Relevance

The main kernel lesson is that MXFP4 GEMM performance depends on shape-specific backend configuration. This PR is not a new kernel implementation by itself; it is a dispatch/tuning layer for selecting AITER kernels from TransformerEngine.
