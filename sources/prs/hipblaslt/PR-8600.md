# [hipblaslt][origami] model changes for MI350P

Open PR #8600 in [ROCm/rocm-libraries](https://github.com/ROCm/rocm-libraries/pull/8600).

**Author:** yenong-amd  
**Updated:** 2026-06-18

## Description

This open PR updates the hipBLASLt/Origami hardware model so MI350P can be differentiated from MI350. The PR body says it implements MI350P bandwidth constants from microbenchmark data and adds PCI chip-ID based differentiation similar to TensileLite.

For wiki use, cite this as open evidence that solution selection is becoming more SKU-specific. A kernel family may be identical at the source level, but the winner selected by Origami can depend on bandwidth constants and hardware identification.
