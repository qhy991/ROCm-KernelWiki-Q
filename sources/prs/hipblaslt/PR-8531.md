# [CK_TILE] launched block size for GEMM occupancy queries

Open PR #8531 in [ROCm/rocm-libraries](https://github.com/ROCm/rocm-libraries/pull/8531).

**Author:** AviralGoelAMD  
**Updated:** 2026-06-17

## Description

This open PR fixes CK Tile occupancy queries in grouped GEMM, grouped-quant GEMM, and Stream-K GEMM kernels. The PR body explains that these kernels queried occupancy with `kBlockSize`, but gfx1250 wave32 launches actually use `kBlockSize/2`; as a result, occupancy came back too low and persistent/Stream-K grids could be undersized. Passing the launched `BlockSize().x` matches universal and flatmm kernels and is a no-op on wave64.

The kernel-design lesson is that occupancy modeling must use launch-time block shape, not only template constants. This matters most for wave32 RDNA paths where compile-time and launched block dimensions intentionally differ.
