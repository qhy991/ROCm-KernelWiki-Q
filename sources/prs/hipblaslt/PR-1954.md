# Physical design change to avoid copying header files

Merged PR #1954 in [ROCm/hipBLASLt](https://github.com/ROCm/hipBLASLt/pull/1954).

**Author:** davidd-amd
**Merged:** 2025-04-21

## Description

> There is no reason to copy header files into the build directory when they aren't being modified during the configuration step. This PR moves files previously copied into a directory inconsistent with the source tree. Instead, we nest them in the appropriate directory and install them into that same directory in the install tree. We also update the impacted header files such that consumption of header files via build or install interface is consistent.

See the PR for full details including code changes and review discussion.
