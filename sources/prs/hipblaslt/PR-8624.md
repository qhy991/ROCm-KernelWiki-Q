# [CK][CK DSL] pass vector sizes as implicit-GEMM arguments

Merged PR #8624 in [ROCm/rocm-libraries](https://github.com/ROCm/rocm-libraries/pull/8624).

**Author:** bartekxk  
**Merged:** 2026-06-19

## Description

This PR addresses incorrect results caused by vector sizes that were too large for the selected implicit-GEMM case. The fix makes vector sizes explicit arguments through CK DSL implicit-GEMM and manifest-runner paths, and updates error calculation and bake-off scripts around that flow.

The kernel-design lesson is that vectorization is part of the problem instance. Implicit GEMM lowering can only safely use wide vector operations when the runner, generated instance, and validation path agree on the vector-size contract.
