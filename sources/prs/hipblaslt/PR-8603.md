# [tensilelite] gfx1250 Subtile PGR=0 WMMA-source WAR hazard fix

Open PR #8603 in [ROCm/rocm-libraries](https://github.com/ROCm/rocm-libraries/pull/8603).

**Author:** WorldofKerry  
**Updated:** 2026-06-19

## Description

This open PR fixes a Subtile correctness issue on gfx1250 when `PGR=0`. The PR body identifies expert scheduling as the source of a write-after-read hazard on WMMA source operands and reports that the gfx1250 cluster Subtile BF16 YAML no longer errors on A0 after the change.

The kernel-design lesson is that Subtile scheduling must preserve operand lifetime around WMMA issue, especially when prefetch/global-read behavior changes the distance between data movement and matrix instructions.
