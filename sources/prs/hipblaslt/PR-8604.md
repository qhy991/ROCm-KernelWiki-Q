# [origami] Subtile-aware heuristic: reject gfx950 BF16 TN subtile kernels

Merged PR #8604 in [ROCm/rocm-libraries](https://github.com/ROCm/rocm-libraries/pull/8604).

**Author:** bethune-bryant  
**Merged:** 2026-06-18

## Description

This PR extends Origami's hipBLASLt solution-selection path so heuristics can see whether a candidate uses the Subtile implementation. It then adds an initial rejection heuristic for a regime where gfx950 BF16 TN subtile kernels are not competitive: `K < 512` with large free dimensions.

The important wiki lesson is that performance tuning lives both in code generation and in solution selection. Adding a kernel family is incomplete until the heuristic model can avoid it for shapes where it loses.
