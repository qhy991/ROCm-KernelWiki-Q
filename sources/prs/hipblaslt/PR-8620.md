# [CK DSL] conv heuristic gemm_k_per_block and 101-feature model update

Merged PR #8620 in [ROCm/rocm-libraries](https://github.com/ROCm/rocm-libraries/pull/8620).

**Author:** cderb  
**Merged:** 2026-06-19

## Description

This PR fixes a CK DSL grouped-convolution heuristic bug where `gemm_k_per_block` was incorrectly represented by `block_size`. The PR states that this confused a GEMM K tile dimension, typically 32-128, with a thread-block size, typically 256, producing a large feature error in tile-efficiency estimates.

It also adds three features, `K_per_C`, `log_gemm_m_n_ratio`, and `log_total_output_tiles`, and updates the supported model artifacts to a 101-feature schema. The PR explicitly calls out retraining across supported architectures and adds first-time gfx950 model coverage.
