# # [TensileLite] Decouple MXFP8 scale DepthU from data DepthU (`ScaleDepthURatio`)

Opened PR #7767 in [ROCm/rocm-libraries](https://github.com/ROCm/rocm-libraries/pull/7767).

**Author:** bnemanich
**Opened:** 2026-05-26

## Description

> Adds a new Solution parameter `ScaleDepthURatio{A,B}` (`R`, valid values `[1, 2]`) that lets the per-DepthU scale fetch cadence run at `1/R` the rate of the data fetch cadence. This unlocks **MT256x256x128 MXFP8 kernels with `R=2`** (data DU=128, effective scale DU=256), which previously could not be generated under the unified-DU model: the larger M/N tile would overflow LDS once scale storage was sized to match data cadence.
