# [hipblaslt][tensilelite] Single-hop next-neighbor StreamK work stealing

Open PR #8442 in [ROCm/rocm-libraries](https://github.com/ROCm/rocm-libraries/pull/8442).

**Author:** jaopaulolc  
**Updated:** 2026-06-18

## Description

Dynamic-queue Stream-K schedulers can leave a workgroup idle when its home queue drains early while neighboring queues still own structural-extra tiles. This PR adds an optional codegen-time `StreamKWorkStealing` parameter so a drained workgroup can steal a single neighboring tile.

The default path remains off, preserving baseline assembly when the feature is not selected.
