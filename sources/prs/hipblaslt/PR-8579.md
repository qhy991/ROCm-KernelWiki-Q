# [hipBLASLt] int8 GEMM alpha=1065353216 crash fix

Open PR #8579 in [ROCm/rocm-libraries](https://github.com/ROCm/rocm-libraries/pull/8579).

**Author:** NaveenElumalaiAMD  
**Updated:** 2026-06-18

## Description

This open PR fixes an int8 GEMM crash where `alpha=1065353216`, the int32 bit pattern for float `1.0`, could lead to `INTERNAL_ERROR`. The PR body attributes the failure to duplicated alpha/beta classification logic: `ConstructTensileProblem` and `updateTensileProblem` could disagree because classification used matrix type or magnitude conversion instead of the true `alphaBetaType`.

The change promotes scalar extraction to a shared helper, removes dead target-type logic, and adds a regression case. Reported validation includes the new four-case test plus complex and `i8_dst_i32` suites on gfx942/gfx950.
