# [GFX1250][CK_TILE] coalesce MX scale16 scale load

Open PR #8566 in [ROCm/rocm-libraries](https://github.com/ROCm/rocm-libraries/pull/8566).

**Author:** AviralGoelAMD  
**Updated:** 2026-06-17

## Description

This open PR fixes a gfx1250 CK Tile MX GEMM memory-access pattern. The PR body explains that `scale16` scale loads were strided across lanes while `scale32` was already coalesced, causing each lane to access a different cache line every K iteration.

The fix puts `scale16` on the same M-fastest descriptor and host pre-shuffle style as `scale32`. The PR describes this as a bijective transpose of data plus descriptor, so scale values are unchanged while lane offsets become contiguous.
