# Use Unified Workspace for FMHA BWD

## Summary
Adapts the Flash Multi-Head Attention Backward (FMHA BWD) host wrappers to use the new unified workspace API from Composable Kernel (CK).

## Changes
- Replaces individual memory tensor arguments (like `dq_acc`) with a single `workspace_ptr`.
- Dynamically allocates device workspace based on `fmha_bwd_launcher::workspace_size` and initializes it via `launcher.prepare_workspace()`.
- Drastically simplifies the kernel launching interface by passing a unified memory blob instead of disjoint gradient accumulators.
