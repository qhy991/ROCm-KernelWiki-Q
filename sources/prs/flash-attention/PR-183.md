# [CK_TILE] FMHA BWD: stream-async workspace prepare

Merged PR #183 in [ROCm/flash-attention](https://github.com/ROCm/flash-attention/pull/183).

**Author:** DDEle
**Merged:** 2026-05-19

## Description

This PR depends on ROCm/rocm-libraries#7331 and moves FMHA backward workspace preparation onto a stream-ordered asynchronous path. The important performance issue is not the math loop itself, but a host/device synchronization point in group-mode backward workspace setup.

The PR replaces a blocking `cu_seqlens.cpu()` style flow with a pinned-host allocation and stream callback lifetime management, allowing workspace metadata preparation to stay ordered with the HIP stream instead of forcing the host to wait before the backward launch can proceed.

## Wiki Use

- `kernel-flash-attention-rocm`: FMHA backward workspace evolution.
- `technique-occupancy-tuning`: host-side synchronization can hide as a kernel performance issue when profiling end-to-end training.
