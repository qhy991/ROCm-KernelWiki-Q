# FMHA BWD: stream-async workspace prepare

## Summary
Eliminates a massive CPU bottleneck during Flash Attention Backward's variable-length (Varlen) sequence processing by moving workspace preparation entirely onto the HIP stream via asynchronous callbacks.

## Changes
- Replaces `prepare_workspace()` with `prepare_workspace_async()`.
- Removes the host-blocking `cu_seqlens.cpu()` Device-to-Host (D2H) copy. The launcher now reads the device `seqstart` directly via async D2H.
- Utilizes PyTorch's `CachingHostAllocator` for pinned memory, which is kept alive via a stream-tail `hipLaunchHostFunc` callback to ensure the pinned buffer is not recycled while async copies are in flight.
