# TDM gather/scatter: reuse one descriptor across chunks to cut in-loop SALU

## Summary
A multi-row `async_tdm_gather` / `async_tdm_scatter` on AMD CDNA4 lowers to N `tensor_load_to_lds` / `tensor_store_from_lds` instructions (chunks). Previously, every chunk rebuilt the full Tensor Descriptor Memory (TDM) descriptor and recomputed its LDS address from scratch.

## Solution
This optimization forces all chunks to share exactly one descriptor. Chunk 0 does the full per-warp LDS-address computation. Chunks 1..N-1 simply use `chunk0_addr + a compile-time-constant byte delta`. 
This causes the compiler to emit `s_add_co_i32` instead of materializing separate per-chunk LDS-address SGPRs, dropping in-loop SALU drastically and sharing one SGPR descriptor tuple across all loads.
