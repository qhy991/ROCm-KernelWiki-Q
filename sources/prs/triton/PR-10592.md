# Pack fp32->bf16 RTZ with v_perm_b32 to relieve VGPR pressure

## Summary
The `fp32 -> bf16` RTZ (Round-to-Zero) path in Triton previously relied on LLVM ISel, which picked a cluster of per-halfword `v_mov_b16` instructions. This inflated VALU usage and VGPR pressure, causing register spilling in the BF16 Flash-Attention kernel on `gfx1250`.

## Solution
This PR manually emits the pack explicitly via the `llvm.amdgcn.perm` intrinsic with the selector `0x07060302`. This selector perfectly isolates the high 16 bits of each `fp32` element (which effectively is RTZ), packing two elements simultaneously without multiple bit-shifts.

## Impact
On a gfx1250 BF16 FA kernel:
- VGPR spills dropped from 1316 bytes to 0 bytes.
- Total VGPR count reduced from 1024 to 982.
- Overall WMMA efficiency skyrocketed from 39% to 73%.
