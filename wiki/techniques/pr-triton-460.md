# Merge AOT Features: Asymmetric SeqLen & Dropout in Triton Flash Attention

## Overview
PR #460 in ROCm Triton merges advanced "Ahead of Time" (AOT) optimizations and features into the Triton `flash-attention.py` kernel. The most significant enhancements include the removal of symmetrical sequence length restrictions—enabling distinct `seqlen_q` (Query) and `seqlen_k` (Key/Value) lengths—and the integration of standard stochastic `dropout` into the attention matrix computation loop.

This enhancement is critical for supporting inference optimizations where sequence lengths can be asymmetrical (e.g. prompt evaluation vs token generation), as well as implementing PyTorch-equivalent SDPA mathematical dropout for the ROCm Triton backend.

## Architectural and Algorithmic Changes

### Asymmetrical Sequence Length Support
Prior to this PR, Triton flash attention assumed queries, keys, and values all shared the same sequence length, defined by a single variable `N_CTX`.
1. **Loop Bounds & Pointer Arithmetic**: The PR decouples this constraint, introducing explicit `seqlen_q` and `seqlen_k` boundaries. This is especially useful for cross-attention mechanisms and generative AI models.
2. **Causal Masking Handling**: Causal masks evaluate conditionally across queries and keys. The PR adapts stage boundaries, making stage transitions `min(seqlen_k, start_m * BLOCK_M)` to ensure precise masking of keys depending on query block traversal.
3. **Padded Blocks**: The `PADDED_BLOCK` condition is refactored to align with `seqlen_k` instead of global `N_CTX`, guaranteeing zero padding loads out-of-bounds on arbitrary `seqlen_k` shapes.

### Integrated Dropout Evaluation
Standard dropout implementations apply uniformly over the $\text{softmax}(QK^T / \sqrt{d})$ probabilities. In Triton block-wise processing:
1. **Philox Random Number Generation**: Uses Triton's pseudo-random number generation (`tl.rand`) seeded with PyTorch-passed `philox_seed` and dynamic `philox_offset` values. Random elements are generated per attention computation element by a local `dropout_mask` function.
2. **Memory Efficiency**: Since allocating the full boolean mask array is $O(N^2)$ and bandwidth-heavy, dropout applies completely *in-kernel*. Specifically, after probabilities `p` are computed, the elements evaluated as dropped out are set to `0.0`.
3. **Scaled Outputs**: To preserve expected statistics within the gradient, valid values are inversely scaled using `acc = acc / (1 - dropout_p)`.
4. **Encoded Softmax Verification**: During testing and debug, PyTorch requires exact dropout states to compare loss. The PR adds an optional `RETURN_ENCODED_SOFTMAX` pointer, serializing $\text{softmax}(QK^T)$ where dropped elements are marked with a flipped sign bit (`-p`). This replicates PyTorch SDPA logic and tests against accurate PyTorch ground-truth.

### Backward Pass Optimizations
The backward algorithms `_bwd_kernel_dk_dv` and `_bwd_kernel_dq` have been heavily refactored:
1. They now explicitly loop and map thread blocks up to `seqlen_q` and `seqlen_k`.
2. Both kernels incorporate dynamic dropout scaling on the gradient `do`. Since the dropped-out weights passed `0` to forward updates, their corresponding backwards gradients zero out matching the same random states re-generated by the deterministic `philox_seed` and `philox_offset`.

## Code Example: Dropout & Scaling Mask
An essential aspect of the PR is handling the PyTorch SDPA (Scaled Dot-Product Attention) alignment in-kernel:

```python
# Extract from _attn_fwd_inner
if ENABLE_DROPOUT:
    philox_offset = batch_philox_offset + start_m * BLOCK_M * seqlen_k + start_n
    keep = dropout_mask(philox_seed, philox_offset, dropout_p, BLOCK_M, BLOCK_N, seqlen_k)
    
    if RETURN_ENCODED_SOFTMAX:
        # Pytorch requires the dropout mask encoded in the sign bits for testing
        tl.store(encoded_softmax_block_ptr, tl.where(keep, p, -p).to(encoded_softmax_block_ptr.type.element_ty))
        
    p = tl.where(keep, p, 0.0)
elif RETURN_ENCODED_SOFTMAX:
    tl.store(encoded_softmax_block_ptr, p.to(encoded_softmax_block_ptr.type.element_ty))
```

## Performance Bounds and Analysis
- **Register Allocation**: Using Philox offsets inside `tl.rand` increases register usage (`VGPR`). The PR ensures offsets are cleanly computed locally using bounds checks.
- **Memory Bounds**: By computing dropout in-kernel, the algorithm avoids an extra $O(N^2)$ round-trip to global memory for random states, strictly keeping the operator computation-bound or block HBM bandwidth-bound instead of memory-bound on the $N \times N$ matrix.
- **Wavefront Sync & Testing**: A minor caveat highlighted in the PR is the dropout accuracy tests (`atol=4e-2`). Scaling and exp-approximations with 16-bit precision drift marginally from PyTorch SDPA references, so the tolerance is broadened.
